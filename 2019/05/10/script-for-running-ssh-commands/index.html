快速多机执行命令工具mrun
发布于 2019-05-10 / 更新于 2019-05-10 / Shell Linux / 热度 0 / 喜欢 0

是否经常需要多台主机执行同一条(组)命令？
对于没有配置免密码认证的主机如何批量多主机执行同一条(组)命令？
今天就来介绍一个脚本工具，可以帮助我们解决以上两个问题，并且可以在此命令工具基础上衍生出更多的命令工具来。

问题

在生产系统的运过程中，我们经常需要在一个集群中的多台主机执行同一个命令（例如查看某个进程服务是否在运行/某个端口是否被监听 或者 执行一个启动脚本来启动某个服务等等）的时候，那么我们可能会写一大段脚本来实现这个功能，很是费时。
如果你是Web开发人员，对脚本编写不熟悉就更是头大。

解决方法

这个问题有很多复杂的解决方法，使用很多大公司开源软件进行管理，不过这需要在每台主机上部署一个服务端服务，保证可以管理主机的系统，适合比较大型的集群综合服务管理。

今天介绍的是一个非常简单的shell脚本，就可以实现批量远程ssh执行特殊命令，它就是 mrun. 当然它只是一个脚本，如果你想要增加功能可以留言，或者自己动手去更新它。代码在 [这里获取(https://github.com/Awkee/auto_ssh_scripts).

下面我们就来介绍一下 mrun命令的使用吧！

mrun命令使用介绍

先来看下这个命令的使用方法，之后再去了解如何部署此脚本工具吧


使用说明:
    mrun [-q -b ] [-e -u username -p password] <ip-list> <cmds>

功能说明:
    此工具帮助我们在多主机批量执行同一任务工作，比如批量开机/关机/批量查看主机参数信息等

参数说明：
    -q    : 安静模式，不会输出分割线信息等提示信息，只会输出命令执行结果
    -b    : 后台运行模式， 命令会提交到后台运行，但是会在最后等待他们执行完成后再退出脚本命令.注意的是输出结果可能会混乱，因为无法控制每个后台进程的完成时间.
    [-e -u username -p password]    :可选参数，默认使用当前登录用户并且可以免认证访问所有主机，如果设置此选项标识需要使用expect命令在指定username下批量执行<cmd>命令,如果username不指定就取默认当前登录用户名
    <ip|ip-list>    : 此参数可以是确定的IP地址如192.168.1.2，也可以是指定的IP范围,示例:192.168.1.2-44,在默认情况下使用的是当前登录用户访问指定IP地址列表,支持逗号指定离散IP列表，也支持减号指定连续IP列表,192.168.2.2-5,192.168.3.2-10,192.168.4.5-20,192.168.5.6
    <cmds>    : 此参数为需要在指定的IP地址下执行的命令语句。

示例:
    mrun 192.168.1.2-23 'hostname;ls -l ~/.ssh/*.pub' : 使用默认的当前登录用户进行免密码认证执行命令
    mrun -u myuser 192.168.1.2-23 'hostname;ls -l ~/.ssh/*.pub'  : 使用myuser用户进行免密码认证执行命令
    mrun -e -u testuser -p testpassword 192.168.1.2-23 'hostname;ls -l ~/.ssh/*.pub' : 使用testuser用户进行自动输入密码方式执行命令

脚本命令的组成和安装介绍

脚本依赖组成分两部分:

使用前提: 在不输入密码的使用情况下，需要将所有需要控制的集群配置ssh免密码认证访问，如果不想配置可以安装expect命令，通过传入参数进行执行即可。

mrun ： shell脚本文件;
auto_ssh_run.exp ：一个expect脚本文件，用于自动登录没有配置免密码认证的主机，需要提前在部署mrun的主机上安装expect命令即可。

部署方式：

只需要将mrun和 auto_ssh_run.exp 两个脚本放到$PATH变量可以访问到的路径下就可以，比如我的放到了${HOME}/bin/目录下。

将 export PATH=$PATH:${HOME}/bin 这一行添加到 .bashrc (如果没有这个文件在查找一下.bash_profile或者.profile文件)环境变量文件，这样以后每次登录就可以直接使用mrun命令了。

$ test -d ${HOME}/bin |||  mkdir ${HOME}/bin
$ export PATH=$PATH:${HOME}/bin
$ wget https://github.com/Awkee/auto_ssh_scripts/archive/latest.tar.gz
$ tar zxvf ./latest.tar.gz
$ cd auto_ssh_scripts-latest
$ chmod +x  auto_ssh_run.exp mrun
$ cp auto_ssh_run.exp mrun ${HOME}/bin
$  mrun 127.0.0.1 'hostname'
 ->命令执行时间:[20190510 13:05:46]<- 
testhost01
>---------------------------end line info:[ip:127.0.0.1 , user:testuser , cmd:hostname]---[分割线]--<

注意:需要给以上两个脚本工具赋权可执行权限的.
